CLI in CI/CD

Use validation as a quality gate before merge or release.

validate-assets --threads 16 --output report.json ^
character:v042:characters/character_v042.usd

Python Publish Hook

Validate assets right before publish to stop bad data early.

from pipeline.core.asset_validator import AssetValidator
validator = AssetValidator()
summary = validator.validate_batch(assets)
if summary.failed: raise RuntimeError("Publish blocked")

Render Farm Preflight

Run preflight checks automatically before dispatching heavy jobs.

if summary.failed > 0:
    notify_team(summary)
    return "blocked"
submit_render_job()

How It Works

Start fast. Validate confidently. Ship clean assets.

1Download the app or install from source.
2Set asset/cache paths and import asset lists.
3Run validation checks for integrity, versions, and USD.
4Review PASS/FAIL reasons and fix issues immediately.
5Export JSON and integrate with your pipeline/CI.
